# DS-086 Default Effect（預設效應）

> 很多人不是做了選擇，而是沿用了沒改的選擇。

## 一句話定義

Default Effect（預設效應）是指人傾向接受系統提供的預設選項，而不是主動更改。

## 核心概念

預設值表面上只是起點，實際上常深刻影響行為。

因為改預設需要注意力、理解和承擔責任，多數人會停在原位。

## 核心邏輯

預設值降低了決策成本，也隱含了「這大概是合理選項」的訊號，於是使用率通常遠高於主動選擇。

## 為什麼重要

預設值看起來像背景，其實常是行為最強的推手之一。

這個模型重要，是因為很多結果不是被選出來的，而是被預設留下來的。

## 使用時機

- 當你在看表單、產品流程、制度設計或政策選項時
- 當群體行為高度集中在某個起始選項時
- 當你要設計更友善或更負責任的選擇架構時

## Key Questions

- 這個結果是主動選擇，還是被動沿用？
- 預設是否在暗示「建議就這樣」？
- 改變預設後，整體行為會大幅改變嗎？
- 這個預設真的符合使用者最佳利益嗎？

## Failure Modes

- 把預設當成中性背景
- 用預設偷渡不利於使用者的選擇
- 忽略退出或更改的摩擦成本
- 只看轉換率，不看長期信任與公平性

## Concept Boundary

預設效應不是所有介面影響，而是人對既定起始選項的偏向接受。

它和強制不同，重點在於不改就沿用的力量。

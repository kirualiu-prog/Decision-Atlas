# DS-065 Path Dependence（路徑依賴）

> 現在之所以長這樣，不只是因為它最好，也因為它先這樣發展了。

## 一句話定義

Path Dependence（路徑依賴）是指過去的選擇會限制現在的選項，讓歷史路徑持續影響未來結果。

## 核心概念

很多制度、產品與行為慣性，不一定是最佳方案，只是因為早期選擇、切換成本與協同效應讓它延續下去。

看不見路徑依賴時，人容易高估現況的「必然性」。

## 核心邏輯

早期決策塑造了結構、標準、習慣與依賴。當切換成本上升，原本路徑就越來越難離開。

## 為什麼重要

路徑依賴能幫你看見現況不是「自然如此」，而是歷史累積的結果。

這對理解制度慣性、技術標準、平台選擇和組織做法都很重要。

## 使用時機

適合用在這些情境：

- 現況看起來明明不最優，卻一直延續
- 你要判斷該順路還是改路
- 團隊過度把歷史選擇當成理所當然

## Key Questions

- 這個模型在這個情境真的適用嗎？
- 它最重要的前提或限制是什麼？
- 如果用這個模型看問題，決策會因此改變什麼？

## Failure Modes

- 把模型套到不適合的問題
- 把模型當成唯一答案
- 記住名詞，卻沒有回到實際判斷

## Concept Boundary

這個模型提供的是一種特定視角，不會單獨取代完整分析。

比較好的用法，通常是和相鄰模型一起交叉檢查。

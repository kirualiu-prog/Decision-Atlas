# DS-075 Value Chain（價值鏈）

> 價值不是憑空出現的，它是在一連串活動裡被創造、轉移或吃掉。

## 一句話定義

Value Chain（價值鏈）是把一個產品或服務從投入到交付的主要活動拆開，分析價值在哪裡被創造與消耗。

## 核心概念

很多優勢不是來自整體抽象策略，而是來自某一段流程做得特別好。

看價值鏈，能幫你找到差異化、成本優勢與真正值得投入的環節。

## 核心邏輯

把主要活動與支援活動拆開，檢查哪些環節提升價值、哪些環節拖慢效率、哪些環節最可能形成優勢。

## 為什麼重要

很多策略討論太抽象，最後不知道優勢到底長在哪個具體環節。

價值鏈分析的重要性，在於它把策略拉回到活動層級，讓你看見價值如何被創造、流失或放大。

## 使用時機

- 當你想找出成本優勢或差異化來源時
- 當你在拆解產品、服務或營運流程時
- 當你要決定哪一段最值得改善、外包或加大投入時

## Key Questions

- 哪個環節真正提升了使用者感受到的價值？
- 哪個環節最耗成本卻沒有對應產出？
- 哪些活動彼此配合後能形成更難複製的優勢？
- 我看到的問題是在產品端、交付端還是支援端？

## Failure Modes

- 把流程拆得太粗，找不到真正差異
- 只看成本，不看價值貢獻
- 把所有環節都視為同等重要
- 找到問題後仍停留在描述，沒有轉成資源配置

## Concept Boundary

價值鏈不是一般流程圖，也不是只做成本盤點。

它關心的是每個活動如何影響價值創造與競爭位置。

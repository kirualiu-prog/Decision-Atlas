# DS-035 Issue Tree（議題樹）

> 問題太大時，不是先急著答，而是先把它拆成能答的小問題。

## 一句話定義

Issue Tree（議題樹）是把一個大問題拆成多個可分析、可驗證、可分工的小問題的結構化工具。

## 核心概念

很多困難問題不是因為沒有能力解，而是因為問題本身太糊、太大、太混。

議題樹能把模糊問題拆到足夠清楚，讓團隊知道應該先證明什麼。

## 核心邏輯

從主問題往下拆分，直到每個分支都能被獨立分析或回答。好的樹會讓你看到缺口、優先順序與可能的解題路徑。

## 為什麼重要

議題樹能把一個太大、太糊的問題，拆成團隊真的可以開始動手的問題。

它不只是分析工具，也很適合溝通、分工與定義研究範圍。

## 使用時機

適合用在這些情境：

- 問題太大，不知道從哪裡開始
- 團隊需要分工處理不同子題
- 你想確認自己是不是問對了問題

## Key Questions

- 這個模型在這個情境真的適用嗎？
- 它最重要的前提或限制是什麼？
- 如果用這個模型看問題，決策會因此改變什麼？

## Failure Modes

- 把模型套到不適合的問題
- 把模型當成唯一答案
- 記住名詞，卻沒有回到實際判斷

## Concept Boundary

這個模型提供的是一種特定視角，不會單獨取代完整分析。

比較好的用法，通常是和相鄰模型一起交叉檢查。
